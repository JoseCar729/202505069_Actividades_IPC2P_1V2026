## Parte 1: Diagnóstico Técnico y Brecha de Impedancia

### 1. La Brecha de Impedancia

La **Brecha de Impedancia** (*Impedance Mismatch*) es la diferencia estructural entre cómo se representan los datos en el paradigma orientado a objetos (C#) y cómo se almacenan en un sistema relacional (SQL Server). EF Core actúa como puente entre ambos dominios mediante las siguientes equivalencias:

 Clase POCO Catedratico -> Tbl_Catedraticos
 Propiedad / Atributo public int Id -> Id INT
 Instancia de Objeto new Catedratico { Id = 1 }`) | → | **Fila / Registro** |

---

### 2. Mitigación de Vulnerabilidades: Inyección SQL

**En ADO.NET tradicional**, el riesgo de Inyección SQL se mitigaba usando **parámetros en `SqlCommand`** a través de `cmd.Parameters.AddWithValue("@param", valor)`. Este mecanismo separa el código SQL de los datos del usuario, impidiendo que un atacante inyecte comandos maliciosos.

**En EF Core**, esta protección es **automática e implícita**. La propiedad que la garantiza es la **parametrización automática de consultas LINQ**: cuando EF Core traduce una expresión LINQ a SQL, siempre convierte los valores variables en parámetros de SQL seguros (`@p0`, `@p1`, etc.), nunca los concatena directamente en el string de la consulta.

---

### 3. Optimización de Infraestructura: `.AsNoTracking()`

Por defecto, EF Core activa el **Change Tracker** (rastreador de cambios) en cada consulta. Este mecanismo mantiene en memoria RAM una "instantánea" de cada entidad recuperada para detectar cambios y poder hacer `SaveChanges()` después.

Cuando el flujo es **exclusivamente de lectura** (por ejemplo, renderizar una lista en pantalla), este rastreo es innecesario y consume memoria RAM sin ningún beneficio.

Al llamar `.AsNoTracking()`, se le indica al contexto que **no cree ni mantenga esas instantáneas en memoria**, lo que:

1. **Libera RAM** del servidor, que es un recurso compartido entre todos los estudiantes y procesos del laboratorio.
2. **Mejora el rendimiento** de la consulta, ya que no se ejecuta la lógica de seguimiento.
3. **Reduce la carga del GC** (Garbage Collector) al haber menos objetos en memoria que limpiar.

Es un acto de "solidaridad computacional" porque un servidor sobreutilizado afecta a todos los usuarios del hardware compartido.

## Parte 2: Desafío de Refactorización de Código

#### Fragmento 1 — El Contexto (`DbContext`)

```csharp
using Microsoft.EntityFrameworkCore;

// Clase POCO (entidad)
public class Catedratico
{
    public int Id { get; set; }
    public string Nombre { get; set; } = string.Empty;
}

// DbContext: unidad de trabajo y acceso a la base de datos
public class UnidadAcademicaContext : DbContext
{
    // Constructor que recibe las opciones de configuración (cadena de conexión, etc.)
    public UnidadAcademicaContext(DbContextOptions<UnidadAcademicaContext> options)
        : base(options)
    {
    }

    // DbSet: representa la tabla Tbl_Catedraticos en la base de datos
    public DbSet<Catedratico> Catedraticos { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        // Mapeo explícito al nombre real de la tabla en SQL Server
        modelBuilder.Entity<Catedratico>().ToTable("Tbl_Catedraticos");
    }
}
```

#### Fragmento 2 — La Consulta LINQ (con `.AsNoTracking()`)

```csharp
using System.Collections.Generic;
using System.Linq;
using Microsoft.EntityFrameworkCore;

public class CatedraticoService
{
    private readonly UnidadAcademicaContext _context;

    public CatedraticoService(UnidadAcademicaContext context)
    {
        _context = context;
    }

    /// <summary>
    /// Devuelve los catedráticos cuyo nombre comienza con "Ing.".
    /// Se usa .AsNoTracking() porque el flujo es solo lectura,
    /// liberando así memoria RAM en el servidor compartido.
    /// </summary>
    public List<Catedratico> ObtenerCatedraticosIngenieros()
    {
        return _context.Catedraticos
            .AsNoTracking()                          // Desactiva el Change Tracker (solo lectura)
            .Where(c => c.Nombre.StartsWith("Ing.")) // EF Core traduce esto a: WHERE Nombre LIKE 'Ing.%'
            .ToList();
    }
}
```
